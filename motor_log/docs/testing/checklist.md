# MotorLog Regression Checklist

Дата ревизии: 12.04.2026

## 1. Подготовка окружения

- [ ] Актуальные зависимости установлены (`flutter pub get`)
- [ ] Файл `.env` присутствует и содержит `WEATHER_API_KEY`
- [ ] Разрешения геолокации выданы приложению
- [ ] Интернет доступен для проверки погодного виджета

## 2. Smoke перед релизом

- [ ] Приложение запускается без ошибок
- [ ] Экран гаража открывается и корректно рендерится
- [ ] Переходы между экранами работают без крэшей
- [ ] Возврат по `Back` работает с каждого экрана

## 3. Гараж и автомобили

- [ ] Добавление авто с валидными данными работает
- [ ] Нельзя сохранить авто с пустыми обязательными полями
- [ ] Обновление пробега валидирует значение (не меньше текущего)
- [ ] После обновления пробега список авто и износ корректно обновляются
- [ ] Удаление авто удаляет карточку из гаража

## 4. Расходники и сервис

- [ ] Для нового авто создаются дефолтные расходники
- [ ] Добавление расходника из справочника работает
- [ ] Добавление кастомного расходника работает
- [ ] Удаление расходника работает
- [ ] Сброс износа создает сервисное событие и обнуляет износ

## 5. Заправки и пробег

- [ ] Нельзя сохранить заправку при пустых полях
- [ ] Нельзя сохранить заправку с пробегом меньше текущего
- [ ] Успешное сохранение заправки показывает подтверждение
- [ ] Пробег авто обновляется при заправке с большим одометром

## 6. Аналитика и калькулятор поездки

- [ ] Сумма общих затрат рассчитывается корректно
- [ ] Детализация по топливу открывается и показывает записи
- [ ] Детализация по сервису открывается и показывает записи
- [ ] Калькулятор поездки пересчитывает литры и стоимость при изменении полей

## 7. Погода и AI

- [ ] При доступном GPS/интернете показываются описание, температура и совет
- [ ] При недоступной погоде показывается fallback-состояние
- [ ] Кнопка обновления погоды перезапускает загрузку
- [ ] AI-чат показывает приветственное сообщение с данными авто
- [ ] AI-чат отвечает на ключевые сценарии: масло, ТО, погода
- [ ] AI-чат корректно обрабатывает вопросы вне сценариев (fallback)

## 8. UI и UX контроль

- [ ] Нет переполнений текста на маленьком экране
- [ ] Нет скачков верстки при загрузке/обновлении
- [ ] `SnackBar`-сообщения информативны и видимы
- [ ] Цвета кнопок/иконок соответствуют теме приложения

## 9. Финальная проверка качества

- [ ] Все unit-тесты проходят (`flutter test`)
- [ ] Нет новых предупреждений, блокирующих релиз
- [ ] Критические сценарии из `test-cases.md` проверены вручную
