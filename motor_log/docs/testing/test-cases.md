# Тест-кейсы MotorLog

## TC-01. Первый запуск приложения с пустой базой

- Приоритет: высокий
- Предусловия: приложение установлено, база данных очищена
- Шаги:
1. Запустить приложение.
2. Дождаться окончания первичной загрузки.
- Ожидаемый результат:
Приложение открывается без падения. На экране отображается заголовок `MotorLog` и сообщение `Ваш гараж пуст. Добавьте машину!`.

## TC-02. Успешное добавление автомобиля

- Приоритет: высокий
- Предусловия: открыт экран пустого гаража
- Шаги:
1. Нажать кнопку `+`.
2. Ввести марку `BMW`.
3. Ввести модель `X5`.
4. Ввести год `2021`.
5. Ввести пробег `45000`.
6. Нажать `Сохранить в гараж`.
- Ожидаемый результат:
Экран добавления закрывается. На экране гаража отображается карточка автомобиля `BMW X5` с пробегом `45000 км`.

## TC-03. Автосоздание стартовых расходников

- Приоритет: высокий
- Предусловия: в гараже уже создан новый автомобиль
- Шаги:
1. Открыть `Детали ТО` у созданного автомобиля.
- Ожидаемый результат:
Открывается экран расходников. В списке присутствуют стартовые позиции: моторное масло, масляный фильтр и тормозные колодки. Для каждой позиции отображается начальный износ.

## TC-04. Сброс износа расходника

- Приоритет: высокий
- Предусловия: открыт экран расходников автомобиля
- Шаги:
1. Нажать иконку обновления у любого расходника.
- Ожидаемый результат:
Износ выбранного расходника меняется на `0%`. Отображается `SnackBar` с подтверждением замены детали.

## TC-05. Удаление расходника с подтверждением

- Приоритет: средний
- Предусловия: открыт экран расходников автомобиля, список не пуст
- Шаги:
1. Нажать иконку удаления у любого расходника.
2. В диалоге подтверждения нажать `Удалить`.
- Ожидаемый результат:
Диалог закрывается. Выбранный расходник исчезает из списка.

## TC-06. Отмена удаления расходника

- Приоритет: средний
- Предусловия: открыт экран расходников автомобиля, список не пуст
- Шаги:
1. Нажать иконку удаления у любого расходника.
2. В диалоге подтверждения нажать `Отмена`.
- Ожидаемый результат:
Диалог закрывается. Состав списка расходников не изменяется.

## TC-07. Удаление автомобиля из гаража

- Приоритет: высокий
- Предусловия: в гараже есть минимум один автомобиль
- Шаги:
1. Нажать иконку удаления на карточке автомобиля.
2. В диалоге подтверждения нажать `Удалить`.
- Ожидаемый результат:
Автомобиль исчезает из списка. При повторном входе в детали данного автомобиля данные больше недоступны.

## TC-08. Отмена удаления автомобиля

- Приоритет: средний
- Предусловия: в гараже есть минимум один автомобиль
- Шаги:
1. Нажать иконку удаления на карточке автомобиля.
2. В диалоге подтверждения нажать `Отмена`.
- Ожидаемый результат:
Диалог закрывается. Автомобиль остается в гараже.

## TC-09. Валидация обязательных полей при добавлении автомобиля

- Приоритет: высокий
- Предусловия: открыт экран добавления автомобиля
- Шаги:
1. Оставить поле `Марка` пустым.
2. Заполнить остальные поля валидными значениями.
3. Нажать `Сохранить в гараж`.
4. Повторить сценарий с пустым полем `Модель`.
- Ожидаемый результат:
Запись не создается. Приложение не падает. Пользователь остается на форме добавления.

## TC-10. Валидация числовых полей

- Приоритет: высокий
- Предусловия: открыт экран добавления автомобиля
- Шаги:
1. Ввести корректные `Марка` и `Модель`.
2. В поле `Год выпуска` ввести нечисловое значение.
3. В поле `Текущий пробег` ввести нечисловое значение.
4. Нажать `Сохранить в гараж`.
- Ожидаемый результат:
Приложение не должно падать. Пользователь должен получить понятную обратную связь о некорректном вводе.

## TC-11. Сохранность данных после перезапуска

- Приоритет: высокий
- Предусловия: в приложении уже создан автомобиль и изменено состояние хотя бы одного расходника
- Шаги:
1. Закрыть приложение.
2. Запустить приложение повторно.
3. Открыть экран гаража.
4. Перейти в `Детали ТО`.
- Ожидаемый результат:
Добавленный автомобиль остается в гараже. Состояние расходников соответствует последним сохраненным данным.

## TC-12. Отображение прогресс-бара при критическом износе

- Приоритет: средний
- Предусловия: в базе есть расходник со значением износа выше `0.8`
- Шаги:
1. Открыть экран расходников автомобиля.
2. Найти расходник с критическим износом.
- Ожидаемый результат:
Прогресс-бар расходника отображается красным цветом.

## TC-13. Несколько автомобилей в гараже

- Приоритет: средний
- Предусловия: база данных очищена
- Шаги:
1. Добавить первый автомобиль.
2. Добавить второй автомобиль.
3. Открыть детали первого автомобиля.
4. Вернуться и открыть детали второго автомобиля.
- Ожидаемый результат:
Оба автомобиля отображаются в гараже. На экране расходников показываются только данные выбранного автомобиля.
